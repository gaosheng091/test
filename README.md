# GitHub Pages 绑定自定义域名教程

本文记录如何把 GitHub Pages 网站(默认地址 `用户名.github.io`)绑定到自己购买的域名上,让别人可以用你自己的网址访问。

---

## 一、前提条件

在开始之前,你需要准备好:

1. **一个已经能正常访问的 GitHub Pages 网站**
   - 例如 `https://gaosheng091.github.io/test/`
2. **一个自己购买的域名**
   - 可以在阿里云、腾讯云、GoDaddy、Namecheap 等平台购买
   - 例如 `example.com`

> 如果还没有域名,可以先跳过本教程,继续使用免费的 `用户名.github.io` 地址。

---

## 二、两种域名类型

在配置前,先搞清楚你要用哪种域名:

| 类型 | 举例 | 说明 |
|------|------|------|
| **顶级域名(Apex)** | `example.com` | 不带 `www`,配置时用 A 记录 |
| **子域名(Subdomain)** | `www.example.com` | 带 `www` 或其他前缀,配置时用 CNAME 记录 |

推荐新手用 **`www` 子域名**,配置更简单也更稳定。

---

## 三、配置步骤

### 第 1 步:在 GitHub 填写自定义域名

1. 进入你的仓库 → **Settings**(设置)
2. 左侧菜单点 **Pages**
3. 找到 **Custom domain**(自定义域名)输入框
4. 填入你的域名,例如 `www.example.com`
5. 点 **Save**(保存)

保存后 GitHub 会自动在你的仓库里生成一个名为 `CNAME` 的文件,记录这个域名。

### 第 2 步:去域名注册商配置 DNS 解析

登录你买域名的平台(阿里云 / 腾讯云 / GoDaddy 等),进入 **DNS 解析 / 域名解析** 设置,根据域名类型添加记录:

#### 情况 A:使用子域名(如 `www.example.com`)—— 推荐

添加一条 **CNAME 记录**:

| 主机记录 / Host | 记录类型 / Type | 记录值 / Value |
|------|------|------|
| `www` | `CNAME` | `用户名.github.io`(例如 `gaosheng091.github.io`) |

#### 情况 B:使用顶级域名(如 `example.com`)

添加 4 条 **A 记录**,分别指向 GitHub 的 IP:

| 主机记录 / Host | 记录类型 / Type | 记录值 / Value |
|------|------|------|
| `@` | `A` | `185.199.108.153` |
| `@` | `A` | `185.199.109.153` |
| `@` | `A` | `185.199.110.153` |
| `@` | `A` | `185.199.111.153` |

> 以上 IP 是 GitHub Pages 官方公布的地址,一般长期稳定。

### 第 3 步:等待 DNS 生效

DNS 解析通常需要 **几分钟到几小时** 才能全球生效(称为 DNS 传播)。

可以用命令检查是否生效:

```bash
# 检查子域名
nslookup www.example.com

# 检查顶级域名
nslookup example.com
```

看到返回的地址指向 GitHub(`github.io` 或那几个 `185.199.x.x` IP)即表示生效。

### 第 4 步:开启 HTTPS(强烈建议)

DNS 生效后,回到 **Settings → Pages**:

1. 等待 GitHub 完成域名验证(会显示绿色对勾 ✅)
2. 勾选 **Enforce HTTPS**(强制 HTTPS)

这样访问会自动加密,地址栏显示 🔒 锁标志,更安全。

---

## 四、验证成功

配置完成后,直接在浏览器访问你的域名:

```
https://www.example.com
```

如果能看到你的网页内容,说明绑定成功!🎉

---

## 五、常见问题

| 问题 | 原因 / 解决办法 |
|------|------|
| 访问显示 404 | DNS 还没生效,再等一会儿;或检查 CNAME/A 记录是否填错 |
| `Enforce HTTPS` 灰色点不了 | 需要等 GitHub 完成证书签发(可能要几小时),之后再来勾选 |
| 保存域名后网站打不开了 | 检查仓库里是否有正确的 `CNAME` 文件,内容应为你的域名 |
| 想取消自定义域名 | 在 Custom domain 框点 **Remove**,并删除仓库里的 `CNAME` 文件 |

---

## 六、小结

绑定自定义域名的核心就是两件事:

1. **在 GitHub 填域名** → 告诉 GitHub "这个域名是我的"
2. **在域名商配置 DNS** → 告诉域名 "请指向 GitHub 的服务器"

两边配置对上,网站就能用你自己的域名访问了。
